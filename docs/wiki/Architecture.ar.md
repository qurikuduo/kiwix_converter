# هندسة Kiwix Converter

## أهداف التصميم

- تحويل أرشيفات ZIM إلى Markdown على مستوى المقالات وملفات JSON جاهزة لأنظمة RAG.
- إبقاء واجهة سطح المكتب قابلة للاستخدام حتى عند إيقاف المهام الطويلة أو انقطاعها.
- حفظ قدر كاف من الحالة داخل SQLite لاستئناف العمل على مستوى كل مقالة.
- التعامل مع `zimdump` وواجهة WeKnora HTTP API كحدود خارجية واضحة بدلاً من إعادة تنفيذ هذه البروتوكولات داخل طبقة الواجهة.

## البنية الطبقية

- `KiwixConverter.WinForms`: غلاف سطح المكتب، والإعدادات، وجداول المهام، وعرض الحالة، وإجراءات المشغل.
- `KiwixConverter.Core.Infrastructure`: مسارات التطبيق، وإعدادات JSON الافتراضية، ومنطق مستودع SQLite وتهيئته.
- `KiwixConverter.Core.Models`: الإعدادات، وصفوف ZIM، وسجلات المهام، ونقاط التوقف، والبيانات الوصفية، وسجلات المزامنة، والسجلات.
- `KiwixConverter.Core.Conversion`: الوصول إلى `zimdump` عبر العمليات الفرعية، واستخراج المحتوى الرئيسي، والتحويل إلى Markdown، والتقسيم، وإعادة كتابة المسارات.
- `KiwixConverter.Core.Services`: تنسيق الفحص، وتنسيق التحويل، والوصول إلى WeKnora API، وتنفيذ المزامنة القابلة للاستئناف.

## الحالة المستمرة

- `settings` تخزن المجلدات و`zimdump` وإعدادات المزامنة مع WeKnora.
- `zim_library` تخزن قائمة ملفات ZIM المحلية.
- `conversion_tasks` و`article_checkpoints` تخزنان تقدم التصدير وحالة الاستئناف.
- `weknora_sync_tasks` و`weknora_sync_items` تخزنان تقدم الرفع وحالة الاستئناف.
- `log_entries` و`weknora_sync_log_entries` تخزنان السجل التشغيلي القابل للبحث.

## تدفق التحويل

1. فحص مجلد kiwix-desktop المهيأ وإجراء upsert لملفات ZIM المكتشفة داخل قاعدة البيانات.
2. استخدام `zimdump` للحصول على بيانات الأرشيف الوصفية وقوائم المقالات و HTML والموارد المرتبطة.
3. استخراج جسم المقالة الرئيسي وتطبيع الروابط والصور وتحويل جزء HTML النظيف إلى Markdown.
4. إنشاء `content.md` و`metadata.json` و`chunks.jsonl` ومجلدات الصور لكل مقالة.
5. حفظ نقاط التوقف لكل مقالة و heartbeat للمهمة حتى تتم إعادة محاولة الجزء المحلي الذي فشل فقط.

## تدفق المزامنة مع WeKnora

1. تحميل قواعد المعرفة من خادم WeKnora والسماح للمستخدم بإعادة استخدامها أو إنشاء قاعدة جديدة.
2. تحميل معرّفات نماذج `KnowledgeQA` و`Embedding` و`VLLM` الحية من `/api/v1/models`.
3. إنشاء KB باستخدام الوصف وchunk size وchunk overlap وخيار parent-child الذي يحدده المستخدم.
4. تطبيق معرّفات النماذج المختارة على تهيئة KB قبل رفع المحتوى.
5. رفع Markdown لكل مقالة كمعرفة يدوية مع حفظ نقاط مزامنة قابلة للاستئناف.

## ملاحظات التشغيل

- إصدار Windows الحالي هو framework-dependent ويتطلب .NET 8 Desktop Runtime.
- البناء المحلي من المصدر يحتاج إلى .NET 8 SDK.
- التطبيق يتعامل مع أكثر من تنسيق لإخراج `zimdump`، لكنه ما زال يفترض وجود `zimdump.exe` صالح للعمل.
